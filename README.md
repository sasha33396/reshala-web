### 🎯 КОРОТКО О ГЛАВНОМ

**«Решала»** — это мой личный "специалист по решению проблем" в мире Linux-серверов, которым я делюсь с вами. Я херачил как конь, чтобы создать этот инструмент и избавить себя и вас от рутины. Это не просто скрипт, это полноценный TUI-фреймворк для управления как одиночным сервером, так и целым флотом.

> **Философия проста:** максимум автоматизации, минимум рутины. Я сделал это, чтобы сервер работал на тебя, а не ты на него.

<br>
  
Я потратил кучу времени, чтобы продумать каждый аспект управления сервером и собрать лучшие практики в удобные модули.

<br>

#### 🌐 Skynet: стань властелином своего флота
>
> Это святой грааль и моя главная гордость. Забудь про `ssh` в 20 окнах. Skynet — твой единый центр управления, который превращает зоопарк серверов в послушную армию.
>
> - **💥 Тотальный контроль:** Управляй десятками серверов как одним. Добавляй, удаляй, выполняй команды на всех сразу. Ты тут босс.
> - **🚀 Телепортация:** Мгновенный вход на любую машину из списка. Ключи? Я сам за них решу.
> - **🤖 Авто-захват:** Натравил Skynet на новый сервер без «Решалы»? Не проблема. Он сам туда десантируется, установится и доложит о готовности.
> - **🛠️ Умные команды:** Не просто `uptime`. Я разделил все команды по категориям, чтобы ты мог точечно бить по проблемам: диагностика, безопасность, система.

<br>

#### 🚦 Шейпер трафика: каждому по заслугам
>
> Еще одна моя гордость. Хватит этой уравниловки, когда один "качальщик" кладёт канал для всех остальных. Этот шейпер — твой персональный вышибала, который каждому выдает строго отмеренный лимит.
>
> - **⚖️ Честное разделение:** Ты выставляешь лимит скорости (например, 10 Мбит/с) **на каждого пользователя**.
> - **😎 Никаких "соседей":** Больше никто не будет страдать из-за того, что кто-то решил скачать все сезоны сериала. Каждый сидит в своем "загоне" и не мешает другим.
> - **🎛️ Простое управление:** Зашел в меню, указал порт и лимиты. Все. Никаких `tc`, `htb` и прочего ада в консоли. Я все сделал за тебя.

<br>

#### 🎛️ Информативный дашборд

*Приборная панель, а не черный экран*

Как только ты входишь в систему, тебя встречает полный обзор состояния сервера:

- **📊 Визуализация:** Наглядные шкалы загрузки CPU, RAM и диска.
- **📈 Оценка производительности:** Запускает официальный **Speedtest от Ookla** и рассчитывает, сколько реальных пользователей выдержит твоя нода.
- **🌍 Статус-панель:** Вся ключевая информация в одном месте — от версии ядра и виртуализации до пинга и состояния Remnawave.
- **⚙️ Кастомизация:** Подключай и отключай виджеты (курс крипты, статус Docker, сетевая активность) и настраивай "профиль нагрузки" дашборда, чтобы снизить потребление ресурсов на слабых VPS.

<br>

#### 🛡️ Комплексный модуль безопасности

*Делаем из твоего сервера крепость*

Я собрал набор утилит "must have" для любого публичного сервера в одном месте.

- **🔥 Firewall (UFW):** Удобный мастер настройки правил с готовыми профилями для стандартных сервисов.
- **👀 Fail2Ban:** Автоматическая блокировка атакующих по логам SSH.
- **🧠 Укрепление ядра (sysctl):** Применение проверенных настроек безопасности на уровне ядра для защиты от спуфинга, smurf-атак и других угроз.
- **📦 Резервные копии:** Создавай и восстанавливай конфигурации безопасности в один клик.

<br>

#### 💿 Жизненный цикл Remnawave

*Полное управление панелью и нодами*

«Решала» — это лучший друг Remnawave. Я автоматизировал все, что мог.

- **🚀 Быстрый старт:** Установка панели, ноды или того и другого вместе в несколько простых шагов.
- **🌐 Управление через Skynet:** Устанавливай ноды на удаленные серверы прямо из интерфейса «Решалы».
- **📜 Все под рукой:** Удобный доступ к логам, перезапуску и управлению установкой.

<br>

#### 🐳 Умное управление Docker

*Держим Docker в узде*

Docker — мощный инструмент, но он любит пожирать дисковое пространство. Этот модуль решает проблему.

- **🧹 Глубокая очистка:** Удаляет неиспользуемые образы, сети, тома и остановленные контейнеры с твоим подтверждением.
- **🎛️ Удобное меню:** Быстрый доступ к логам, статистике (`docker stats`), а также к старту, стопу и перезапуску любого контейнера.

### Если не работает команда reshala выполни команды ниже

```bash
# 1. Убрать алиас из текущей сессии
unalias reshala 2>/dev/null

# 2. Убрать из .bashrc
sed -i "/alias reshala=/d" /root/.bashrc

# 3. Создать умный враппер вручную
cat > /usr/local/bin/reshala << 'EOF'
#!/bin/bash
TARGET="/opt/reshala/reshala.sh"
if [[ $EUID -eq 0 ]]; then
    exec "$TARGET" "$@"
elif command -v sudo >/dev/null 2>&1; then
    exec sudo "$TARGET" "$@"
else
    exec "$TARGET" "$@"
fi
EOF
chmod +x /usr/local/bin/reshala

# 4. Запуск
reshala
```
